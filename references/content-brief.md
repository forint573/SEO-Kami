# Content Brief + Competitor-GAP Map

Turn ONE target query into a writer-ready BRIEF, plus a deterministic competitor-GAP map. SEO-Kami produces the PLAN; it does NOT write the article. The user's the-sonnopus-humanizer skill plus the relevant translation skill write the actual copy. Every substantive claim carries an evidence tier per references/evidence-tiers.md: PROVEN, CORRELATED, CONSENSUS, SPECULATIVE.

## What this produces (and does not)

- PRODUCES: a single per-query BRIEF (intent, titles, meta, outline, direct-answer block, term placement, link slots, angle, schema slot, E-E-A-T checklist) and a GAP MAP for the topic.
- DOES NOT: write or ghostwrite the article, invent copy, or fabricate metrics. Handoff is the brief; authoring is the user's the-sonnopus-humanizer + translation skills.
- One brief = one target query = one page type. If the live SERP mixes intents, the query is too broad for a single URL — split it (see references/keyword-free-signals.md). (CONSENSUS)

## Brief structure

A brief is the ordered set of sections below. Build it in this order — intent gates every later choice. Drive demand-side inputs from references/keyword-free-signals.md (Autocomplete, PAA, Related Searches, live SERP read, and your own Google Search Console export); never plug in a search-volume or difficulty number, because no reliable FREE source exists. (PROVEN)

## Target query + intent

- State the primary term verbatim and classify ONE dominant intent: informational / navigational / commercial-investigation / transactional. Verify the classification against the LIVE SERP — the SERP is the authority on intent, not your guess. (CONSENSUS) See references/keyword-free-signals.md for the classification + page-type mapping.
- Record the SERP features present (featured snippet, PAA, image/video pack, local pack, AI Overview). These set both the click value and which extractable assets the brief must include. (PROVEN)
- Prioritization signals to note inline (NO volume/difficulty — none is reliably free): SERP-feature click value; Google Search Console striking-distance position (your own impressions/position ~8-20); first-party business fit; topical authority. (PROVEN that no free volume exists; CONSENSUS on the signal set.)
- If the query triggers an AI Overview, expect ~83% zero-click and CTR ~37% below AIO-absent (CORRELATED, Seer 53 brands / 5.47M queries). Favor transactional/comparison/deep intent where a click still converts; pursue pure-informational targets for surfacing and citation, not traffic. See references/measurement-zero-click.md.

## Title options

- Give 2-3 options. Front-load the primary term (leftmost weighting; the term users scan for). (CONSENSUS)
- Be honest about the pixel limit: Google truncates the SERP title at roughly 580px (~60 characters as a rough proxy, not a hard cutoff — width is per-character). State the rough limit; do not promise an exact character count. (CONSENSUS)
- Differentiate the options by angle, not by keyword stuffing: e.g. one literal, one benefit-led, one with a credibility/specificity hook (year, number, named method). Google may rewrite your title; a clear, accurate title is rewritten less often. (CONSENSUS)
- The chosen title and the H1 may differ; the H1 can be longer and need not obey the pixel limit.

## Meta-description suggestion

- One suggestion, ~150-160 characters as a rough target. State plainly that meta descriptions are NOT a ranking factor (PROVEN) and that Google rewrites them on most queries — write it to earn the click when shown, not to rank.
- Include the primary term once, naturally, and a concrete reason-to-click (the specific value or differentiator). Avoid filler and duplicate boilerplate across pages.

## H2/H3 outline from real questions

- Build the outline from REAL questions, not invented headers. Source them from PAA, Autocomplete prefixes, and Related Searches per references/keyword-free-signals.md. Each PAA item is both a question-shaped heading and an answer-feature opportunity. (CONSENSUS)
- Make H2/H3 mirror the user's actual phrasing (question-shaped where the source is a question). This is good SEO and raises extraction odds for snippets/PAA/AI answers — see references/geo-aeo.md sections 4 and 8. (PROVEN-leaning)
- One H1, descriptive and unique. Logical H2 > H3 nesting; do not skip levels for styling. Map each major H2 to one sub-intent; if two H2s answer the same sub-intent, merge them.
- Mark which headings should carry an immediate direct answer underneath (the PAA/voice/featured-snippet pattern: question heading + concise direct answer).

## Direct-answer block (AEO/GEO)

- Specify ONE self-contained ~40-60 word direct answer placed near the top of the page (after the H1/intro, before deep body). It must stand alone — quotable without surrounding context — because that is what extraction engines lift. (PROVEN-leaning; it is good SEO.) See references/geo-aeo.md section 4.
- It should answer the primary query plainly, include one verifiable specific (a number, date, or named source) where honest, and avoid hedging that breaks extractability.
- Note in the brief: this block targets featured snippets / PAA (AEO) AND citability inside generative answers (GEO) — same content, both surfaces, same index. It is NOT a schema trick and NOT a guaranteed citation; extraction is probabilistic and query-dependent. (PROVEN that GEO/AEO is SEO; CONSENSUS on probabilistic citation.)

## Secondary-term placement table

Secondary/related terms come from Autocomplete, Related Searches, "People also search for", and competitor on-page extraction (references/keyword-free-signals.md). Place them by role. NO volume, difficulty, or any estimate column — none is reliably free. (PROVEN)

| Secondary term | Placement | Note |
|---|---|---|
| [variant / synonym] | Intro | Natural early reinforcement; once, not stuffed |
| [sub-topic phrase] | H2 | Becomes a section heading mirroring real phrasing |
| [long-tail / qualifier] | Body | Inside the relevant section, in context |
| [entity / related concept] | Body | Supports topical coverage + entity clarity |
| [adjacent question] | H3 | Sub-question under its parent H2 |

- Rule: each term earns a placement only if a real user phrasing or sub-intent backs it. Do not place a term to "hit a keyword"; place it because the section genuinely covers it. (CONSENSUS)

## Internal-link slots + external-citation slots

- INTERNAL-LINK slots: name the specific existing pages this brief should link to, with intended anchor text. Prioritize commercial/conversion pages and topical-cluster siblings/hub pages so authority and intent flow correctly. Descriptive anchors, not "click here". (CONSENSUS)
- EXTERNAL-CITATION slots: name high-trust sources to cite inline (primary docs, official statistics, named studies, standards bodies). Verifiable, named, dated citations support E-E-A-T Trust and raise citability in answer engines. See references/content-eeat.md and references/geo-aeo.md section 4. (PROVEN-leaning)
- Specify each slot as: location in the outline -> target URL/page -> anchor/citation text. Leave the slot as a visible placeholder if the exact source is TBD — never fabricate a citation.

## Differentiation / angle

- State the ONE thing that makes this page non-commodity: a unique first-hand experience, original data/test, proprietary POV, or specificity a recycled summary cannot fake. Without it, the page is the commodity content core updates demote. (PROVEN that commodity/recycled content is demoted.) See references/content-eeat.md.
- The angle is the Experience signal made concrete: original photos/screenshots, a documented test ("we ran X for N months"), named conditions, first-party numbers — things only someone who did the work would know. (CONSENSUS) This is what survives the answer-surface era; AI-spun summaries cannot reproduce it.
- Write the angle as a one-sentence promise the writer must deliver on. If the brief cannot name a real differentiator, that is a signal NOT to create the page (or to gather the first-hand input first).

## Schema slot

- Route every structured-data decision through references/schema-2026.md and copy from assets/schema-templates.json. Name the still-earning type(s) that fit the page: Product, Review/AggregateRating, Article, Recipe, Video, Event, Organization, Person, LocalBusiness, Breadcrumb. (PROVEN status table.)
- Use JSON-LD only; supply the minimum required properties per type (see references/schema-2026.md "Required properties"). Omitting a required property silently forfeits eligibility. (PROVEN)
- FAQPage / HowTo: NEVER list as a ranking or rich-result lever. FAQPage rich results were removed ~2026-05-07 (still valid markup, zero enhancement outside authoritative gov/health); HowTo earns no rich result on any surface. (PROVEN) Include an FAQ section ONLY if it is genuinely useful UX for the reader — and if so, mark it CONSENSUS as a UX choice, not a rich-result play. Schema is not required for AI features and is not an AI shortcut. (PROVEN)

## E-E-A-T checklist

A design target for genuinely trustworthy content — there is no "E-E-A-T score". (PROVEN) See references/content-eeat.md. The brief must specify:

- AUTHOR: named real byline linked to an author page with relevant credentials and `sameAs` profiles (match credential type to topic; formal for YMYL, lived for hobby/consumer). (PROVEN/CONSENSUS)
- FIRST-HAND PROOF: the concrete Experience asset the writer must include — original media, test data, named specifics (the angle, made shippable). (CONSENSUS)
- CITATIONS: inline links to primary/authoritative sources (the external-citation slots above), accurate and dated. (PROVEN/CONSENSUS)
- TRUST hygiene: visible publish/update date reflecting a real revision, transparent sourcing, reachable About/contact, no deception. For YMYL, add a named reviewer and conservative dated claims. (PROVEN)

## Competitor-GAP sub-section

(CONSENSUS as a method.) Produces the GAP MAP that feeds the calendar in references/content-strategy.md. The discipline: derive everything DETERMINISTICALLY; guess nothing.

### Identify competitors
- Competitors = the actual top organic results for the target topic on the live SERP (not who the business names as rivals). Read them directly. (CONSENSUS)

### Page inventory — deterministic only
- Derive each competitor's page inventory from their `sitemap.xml` (and any nested sitemap-index children). This is observed, not modeled: a URL list and its `lastmod` dates. (CONSENSUS)
- FORBIDDEN columns: Est. Pages, Est. Traffic, Est. Volume, Difficulty, Local Search Volume, or any modeled count. No reliable FREE source provides these — never invent them. (PROVEN) Report only what the sitemap literally contains: URLs, paths, and `lastmod` timestamps.
- From the URL/path structure infer topic coverage; from `lastmod` infer staleness. Both are deterministic reads of the file, not estimates.

### Three gap axes

| Axis | Definition | Derived from |
|---|---|---|
| 1. Topics-they-cover-you-lack | Sitemap topics/paths present for competitors, absent on your site | Sitemap path diff (theirs vs yours) |
| 2. Blue-ocean | High-value sub-topics NO competitor covers well — the non-obvious axis | PAA/Autocomplete demand (references/keyword-free-signals.md) minus competitor sitemap coverage |
| 3. Competitor weakness | Thin or outdated pages to outdo | Stale `lastmod` + shallow on-page (title/H-structure read) on a ranking URL |

- Axis 2 (blue-ocean) is the prize: real user questions (from PAA/Autocomplete) that the SERP answers poorly or not at all. It is non-obvious precisely because no competitor URL signals it. (CONSENSUS)
- Axis 3 uses the same on-page extraction as references/keyword-free-signals.md (titles, H1-H3, slugs) cross-referenced with old `lastmod` — qualitative thin/stale judgment, never an inferred-traffic claim.

### Per-gap action plan (feeds the calendar)

For each gap, output one row -> hands directly to references/content-strategy.md:

| Gap | Axis | Why it matters (intent + first-party fit) | Action -> brief? |
|---|---|---|---|
| <topic/URL gap> | 1/2/3 | <intent class; click value; business fit; striking-distance if any> | <commission a brief (above) / refresh existing page / skip> |

- Prioritize gaps by the same no-volume signal set: SERP-feature click value, your Google Search Console striking-distance, first-party business fit, topical authority — NOT by any invented volume. (PROVEN no-free-volume limit; CONSENSUS on signals.)
- Each "commission a brief" row re-enters the top of this file as a new TARGET QUERY; "refresh" rows update an existing page's brief. The loop closes into the calendar in references/content-strategy.md.
- Measure outcomes per references/measurement-zero-click.md (impressions, AI-citation share, brand-mention growth), not raw session growth.
